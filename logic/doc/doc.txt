# Architecture (Marketplace-Oriented)

1️. Node Agent (Runs on contributor machines) Each participant runs:

 - A small Go binary (5–20MB)

 - Registers to your coordinator

 - Reports: CPU cores, RAM, GPU type, Availability

 - Executes sandboxed jobs


2️. Lightweight Job Orchestrator (Central Service): Instead of EKS, run:

 - 1 small VPS (DigitalOcean / Hetzner / AWS Lightsail)

 - Docker only (no K8s)

 - PostgreSQL

 - Redis (for queue)

  Cost: $10–20/month.


3️. Containerized Job Execution On each node use:

 - containerd

 - Or plain Docker

 - Or Firecracker (advanced stage)

 - When a job arrives: Pull job container => Execute => Return result => Destroy container ( No need for Kubernetes scheduler ).

# Technology Stack (Low Cost)

 - Coordinator: Go API server, PostgreSQL, Redis queue, Docker

 - Node: Go agent, Docker, gRPC or WebSocket connection

 - Billing: Wallet system

 - Meter usage by: CPU time, GPU time, RAM time
